# 投票フロー・UI設計（Helios Votingを参考）

UI設計はHelios Voting（`github.com/benadida/helios-server`）のフローを参考にする。

## 画面遷移

```
Step 1: 選挙トップ          /elections/[id]
  ┌─────────────────────────────────────────────┐
  │ 【第○区 衆議院予備選挙】                        │
  │                                             │
  │ 投票期間: 2026/4/1 9:00 〜 2026/4/7 20:00    │
  │ ステータス: 投票受付中                          │
  │                                             │
  │ 候補者:                                       │
  │  ・山田太郎 — プロフィール概要...               │
  │  ・佐藤花子 — プロフィール概要...               │
  │                                             │
  │  [マイナンバーカードで認証して投票する]           │
  └─────────────────────────────────────────────┘

Step 2: 認証                /auth → デジタル認証アプリ → /auth/callback
  ┌─────────────────────────────────────────────┐
  │ マイナンバーカードで本人確認中...                │
  │                                             │
  │  (デジタル認証アプリにリダイレクト)              │
  │  → スマホでマイナンバーカードをかざす           │
  │  → 4桁PINを入力                              │
  │  → コールバックで戻る                          │
  └─────────────────────────────────────────────┘

Step 3: 資格確認            （サーバーサイド処理）
  - 住所から選挙区を判定
  - 選挙の対象選挙区と一致するか確認
  - 不一致の場合 → 「この選挙の投票資格がありません」画面
  - 一致の場合 → Step 4へ

Step 4: 投票選択            /elections/[id]/vote
  ┌─────────────────────────────────────────────┐
  │ 投票する候補者を選んでください                   │
  │                                             │
  │  ○ 山田太郎                                   │

  │                                             │

  │                                             │
  │  [確認画面へ]                                  │
  │                                             │

  └─────────────────────────────────────────────┘

Step 5: 確認・暗号化        /elections/[id]/vote（確認ステップ）
  ┌─────────────────────────────────────────────┐
  │ あなたの投票内容を確認してください               │
  │                                             │
  │  選択した候補者: 【山田太郎】                    │
  │                                             │
  │  [暗号化して投票する]   [選び直す]              │
  │                                             │
  │  ⏳ 投票内容を暗号化しています...               │
  │  （ブラウザ内で暗号化。サーバーには平文を送らない）│
  └─────────────────────────────────────────────┘

Step 6: 完了・Ballot Tracker  /elections/[id]/vote/complete
  ┌─────────────────────────────────────────────┐
  │ ✅ 投票が完了しました                          │
  │                                             │
  │ あなたの Ballot Tracker:                      │

  │  [選挙トップに戻る]                             │
  └─────────────────────────────────────────────┘
```

## Heliosから採用する設計パターン

1. **ブラウザ内暗号化**: サーバーは平文を一切見ない
2. **Ballot Tracker**: 暗号化投票のハッシュ値で投票記録を確認可能
3. **候補者ランダム表示**: セッションごとにランダム化
4. **監査投票（Audit Ballot）**: 将来的に実装を検討
5. **掲示板（Bulletin Board）**: 全暗号化投票の一覧を公開

## 投票フロー（技術的シーケンス）

```
[投票者] → 投票画面にアクセス
  → 「マイナンバーカードで認証」ボタン（モック時はユーザー選択画面）
  → OpenID Connect 認可コードフロー開始
  → [認証完了] IDトークン + userinfo から sub, 住所を取得
  → subのハッシュで一人一票を確認
  → 住所から選挙区を判定 → 投票資格を確認
  → [資格あり] 候補者一覧を表示（表示順ランダム化）
  → 候補者を選択 → 確認画面
  → 投票内容をクライアント側で暗号化（sealed box）
  → Ballot Tracker（暗号文のハッシュ）を生成
  → サーバーに暗号化データを送信
  → 有権者名簿の投票済みフラグを更新
  → Ballot Trackerを投票者に表示
  → 投票完了画面
```
